Vertical Slice Implementation with Claude Code

1,247 views v1.2.0 Updated Jan 15, 2026

A comprehensive prompt for implementing complete vertical slices in .NET applications using the Wolverine framework. This prompt guides Claude Code through the entire feature implementation process, from creating request/response models to writing tests, ensuring consistency across your codebase.

User Story Implementation Claude Code .NET Wolverine Vertical Slice
Prompt Content
You are implementing a vertical slice for a .NET application using the Wolverine framework with a vertical slice architecture. Your task is to implement a complete feature from API endpoint to database. [USER_STORY] Follow these implementation guidelines: 1. **Create the Request/Response models** - Define strongly-typed request and response records - Include all necessary validation attributes - Use nullable reference types appropriately 2. **Implement the Handler** - Create a Wolverine handler in the appropriate feature folder - Follow the pattern: Features/[FEATURE_NAME]/[ACTION]Handler.cs - Use dependency injection for repositories and services - Include proper error handling and logging 3. **Add the API Endpoint** - Map the endpoint in the appropriate endpoint group - Use proper HTTP methods (GET, POST, PUT, DELETE) - Include OpenAPI documentation attributes - Add authorization attributes if needed 4. **Create Database Migrations (if needed)** - Generate EF Core migrations for schema changes - Review the migration before applying 5. **Write Tests** - Create unit tests for the handler logic - Create integration tests for the full slice - Use xUnit and NSubstitute for mocking After implementation, verify: - The code compiles without warnings - All tests pass - The API responds correctly when called [ADDITIONAL_CONTEXT]
